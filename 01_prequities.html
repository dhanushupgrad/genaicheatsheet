GenAI Prerequisites Guide

What is a Token?

A token is the most fundamental data unit in a text document, essential for enabling AI to understand and process information. Tokenization refers to breaking down larger texts into smaller, manageable pieces called tokens.

A token is the basic unit of text that a language model processes. AI models break text into tokens (chunks) so they can compute probabilities for the next piece of text.

A token can be:
  • A word (e.g., "cat")
  • A subword (e.g., "play" + "ing")
  • A punctuation mark (e.g., ".")
  • Even a single character depending on the tokenizer
Example: I love machine learning!

Possible tokenization (depends on tokenizer used):
  • Word-level: ["I", "love", "machine", "learning", "!"] → 5 tokens
  • Subword-level (common in GPT-like models): ["I", "love", "machine", "learn", "ing", "!"] → 6 tokens
  • Character-level: ["I", " ", "l", "o", "v", "e", " ", "m", "a", "c", "h", "i", "n", "e", " ", "l", "e", "a", "r", "n", "i", "n", "g", "!"] → 24 tokens

Why Tokens Matter

  • Context Window: Models can only handle a fixed number of tokens (e.g., 4k, 16k, 128k).
  • Cost & Speed: More tokens → higher compute cost and slower response.
  • Precision: Tokenization affects how well the model understands rare words, names, or technical terms.

1. Context Window/Prompt

The context window is the maximum number of tokens that you can provide to it at one time. This is how much text the model can "remember" at once.

This includes:
  • System instructions
  • User messages
  • Assistant replies

If the conversation exceeds the window:

  • Older tokens are forgotten
  • Reasoning may degrade
  • Small context window → misses details
  • Large context window → accurate summary
Example: "The Cat sat on the mat"
Let us assume, we have 6 tokens in the above sentence.
GPT-4o can process 128K tokens at a time

2. Temperature

Temperature is a parameter that controls the randomness in the model's output. Low values of temperature make the text more predictable and consistent, while high values let more freedom and creativity into the mix, but can also make things less consistent.

Think of it as a creativity knob.

  • Low temperature (≈0.0–0.3) → predictable, factual, conservative
  • Medium (≈0.7) → balanced
  • High (≈1.0–1.5) → creative, surprising, sometimes chaotic
from transformers import pipeline
import warnings
warnings.filterwarnings('ignore')

generator = pipeline("text-generation", model="gpt2")

output = generator(
    "Once upon a time",
    temperature=0.7,
    do_sample=True  # Enables sampling (required for temp/top-p)
)

print("Model Response:", output[0]['generated_text'])

3. Top-p (Nucleus Sampling)

"Top_p" or nucleus sampling is a setting that decides how many possible words to consider. Top-p chooses the smallest set of tokens whose total probability ≥ p.

Example:
If you ask for a title for an adventure book, with a top-p of 0.5, the model might come up with: "The Mystery of the Blue Mountain."

Top_p = 0.9: This includes a lot more words in the choice, allowing for more variety and originality.
For the same adventure book title and a top-p of 0.9, the model might create: "Voices from the Abyss: A Portrait of the Brave."
✓ Best default for most creative tasks
✓ More stable than top-k

Mixing Temperature and Top_p: What Happens?

Combining "temperature" and "top_p" can give a wide range of text styles.

  • A low temperature with a high top-p can lead to coherent text with creative touches.
  • A high temperature with a low top-p might give you common words put together in unpredictable ways.

What About Low Temperature and High Top_p?

Here, answers are usually logical and consistent because of the low temperature, but they can still have rich vocabulary and ideas due to the high top-p. This setup is good for educational or informative texts where clarity is crucial, but you also want to keep the reader's interest.

And High Temperature with Low Top_p?

This opposite setting often results in texts where sentences may make sense on their own but as a whole seem disconnected or less logical. The high temperature allows more variation in sentence building, while the low top-p limits word choices to the most likely ones. This can be useful in creative settings where you want unexpected results or to spark new ideas with unusual concept combinations.

4. Max Tokens

Max tokens sets the maximum length of the model's response.

Important: Tokens ≠ words
1 token ≈ ¾ of an English word (roughly)
The model stops generating once it hits this limit, even mid-sentence.
Example - Prompt: Explain gravity.

Max tokens = 20 → "Gravity is a force that pulls objects with mass toward each other."
Max tokens = 200 → Full multi-paragraph explanation

5. Stop Sequences

If you see this text, stop generating immediately.

Commonly used to:
  • End at "\nUser:"
  • Stop at "###"
  • Enforce format boundaries
from transformers import AutoModelForCausalLM, AutoTokenizer
import warnings
warnings.filterwarnings('ignore')

model_name = "gpt2"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

prompt = "Write a short poem about the ocean.\nUser:"
inputs = tokenizer(prompt, return_tensors="pt")

output = model.generate(inputs["input_ids"], max_length=100)
text = tokenizer.decode(output[0], skip_special_tokens=True)

# Stop at "\nUser:" or "###"
for stop_seq in ["\nUser:", "###"]:
    if stop_seq in text:
        text = text.split(stop_seq)[0]

print("Response:", text)

6. Repetition Penalty / Frequency Penalty

Reduces the likelihood of repeating the same tokens. The higher the penalty, the more the model avoids repeating words/phrases. Useful for making outputs more diverse and natural.

Example - Prompt: Write a paragraph about AI.

Penalty = 0.0 → "AI is powerful. AI is evolving. AI is everywhere."

Penalty = 1.2 → "Artificial intelligence is powerful, evolving rapidly, and shaping many industries." 
from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "gpt2"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

prompt = "Write a paragraph about AI."
inputs = tokenizer(prompt, return_tensors="pt")

# Try with different repetition penalties
output_no_penalty = model.generate(
    inputs["input_ids"],
    max_length=50,
    repetition_penalty=1.0  # No penalty
)

output_with_penalty = model.generate(
    inputs["input_ids"],
    max_length=50,
    repetition_penalty=1.2  # Penalize repetition
)

print("Penalty = 1.0 →", tokenizer.decode(output_no_penalty[0], 
                                          skip_special_tokens=True))
print("Penalty = 1.2 →", tokenizer.decode(output_with_penalty[0], 
                                           skip_special_tokens=True))
Output Example:

Penalty = 1.0 → Write a paragraph about AI. The first thing you need to do is to write a paragraph about AI. The first thing you need to do is to write a paragraph about AI. The first thing you need to do is...

Penalty = 1.2 → Write a paragraph about AI. The first thing you need to do is write the following: "I am an engineer, and I..."
