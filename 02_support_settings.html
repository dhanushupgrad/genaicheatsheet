1. Beam Search

Traditional search methods like depth-first and breadth-first searches have limitations, especially when it comes to efficiency and memory usage. Where for that we have Beam Search.

Beam Search is a heuristic-based approach that offers a middle ground between these traditional methods by balancing memory consumption and solution optimality.

Beam search is a graph-based search technique that explores only a limited number of promising nodes (a beam). The beam width, which limits the number of nodes stored in memory, plays a crucial role in the performance and accuracy of the search.

Heuristic Techniques

These techniques help find the most efficient path from a starting point to a goal. Heuristic search operates within the search space of a problem to find the best or near-optimal solution using systematic algorithms.

Unlike brute-force methods, which exhaustively evaluate all possible solutions, heuristic search leverages heuristic information to guide the search toward more promising paths.

How it works (intuition)

  • At each step, the model explores several promising continuations
  • It scores full sequences, not just the next token
  • Finally, it selects the highest-probability sequence overall

Typical use cases

  • Machine translation
  • Speech recognition
  • Structured or factual generation

Example Code: Beam Search with T5

from transformers import AutoModelForSeq2SeqLM, AutoTokenizer

model_name = "t5-small"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSeq2SeqLM.from_pretrained(model_name)

input_text = "Translate English to German: The house is wonderful."
inputs = tokenizer(input_text, return_tensors="pt")

# Beam search with 3 beams
outputs = model.generate(
    **inputs,
    num_beams=3,           # number of beams
    early_stopping=True    # stop when all beams reach EOS
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
Output: Das Haus ist wunderbar.

2. Seed (for reproducibility)

Randomness in sampling (temperature, top-p, etc.) means outputs vary. Setting a seed ensures you get the same result every run.

A seed initializes the model's random number generator so that sampling decisions are repeatable.

Why it matters

Same prompt + same seed + same settings = same output

Crucial for debugging, evaluation, and experiments.

Common use cases

  • Research and benchmarking
  • A/B testing prompt changes
  • Regenerating exact outputs for review or auditing

Example Code: Setting Seeds

import torch
import random
import numpy as np

# Set seeds for reproducibility
torch.manual_seed(42)
np.random.seed(42)
random.seed(42)

# Now every run will give the same sampled output
outputs = model.generate(
    **inputs,
    do_sample=True,    # enable sampling
    top_k=50,
    max_length=40
)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
Output: Das Haus ist wunderbar, die heutige Tradition zu haben, und die ganze Welt durch

3. Batch Size & Sampling Strategy

The number of prompts (or generations) processed simultaneously.

Why it matters

  • Larger batches improve throughput and GPU utilization
  • Smaller batches reduce latency and memory use

Comparison Table

Batch Size Advantage Downside
Small Faster response time Less efficient
Large Higher throughput More memory, higher latency
